SyncTools for Unreal Engine 🚀

概述

SyncTools for Unreal Engine 是一款专为 Unreal Engine 环境中复制和粘贴详细场景数据而设计的专用插件，旨在简化这一流程。只需一键，即可从一个场景导出静态网格 Actor（包含其完整的变换数据、材质和属性），并将其导入到另一个场景中。这显著提升了关卡设计和场景合成工作流程的效率。

🛠 Unreal Engine 安装与使用

1️⃣ 插件安装

🔹 下载插件文件 Synctools_UE.py
🔹 将文件复制到您的 Unreal Engine Python 脚本目录
🔹 重启 Unreal Engine
🔹 安装成功后，您将在主菜单栏中找到 "SyncTools"

2️⃣ 工作流程

📋 复制操作

✅ 在场景中选择要复制的静态网格 Actor
✅ 从 SyncTools 菜单点击 "复制"

📂 粘贴操作

✅ 切换到目标场景
✅ 从 SyncTools 菜单点击 "粘贴"
✅ 插件自动导入最新的缓存文件，并在导入成功后自动清理

📝 版本兼容性

✅ 官方支持：Unreal Engine 4.26 及更高版本

✅ 官方支持：Unreal Engine 5.04 及更高版本

📊 数据兼容性

✔ 完全支持：

🔹 静态网格 Actor
🔹 变换数据（位置、旋转、缩放）
🔹 Actor 标签和名称
🔹 顶点颜色
🔹 LOD 设置

⚠ 部分支持：

🔸 材质（基础属性）
🔸 纹理（需要匹配路径）
🔸 碰撞网格（导出时过滤）

🔸 动画和关键帧（基础属性）

❌ 不支持：

❌ 骨骼网格
❌ Blueprint Actor
❌ 粒子系统
❌ 地形数据
❌ 动态对象
❌ 灯光 Actor

⚡ 重要说明

1️⃣ 缓存管理

🗑 操作成功后自动清理
📁 缓存位置：Documents/cache/

2️⃣ 资产处理

🎯 自动过滤碰撞网格
🏷 保留原始 Actor 名称和标签
📍 保持精确的变换数据

3️⃣ 最佳实践

✔ 仅用于静态网格 Actor
✔ 确保缓存文件有足够的磁盘空间
✔ 保持源和目标 Unreal Engine 版本兼容

4️⃣ 性能

⚡ 针对快速复制粘贴操作优化
📉 最小化临时存储使用
🧹 高效的清理流程

5️⃣ 数据交换

🔄 FBX 格式用于网格数据
📜 JSON 格式用于变换数据
🔧 自动文件管理

🎯 总结

SyncTools for Unreal Engine 专为在 Unreal Engine 环境中提供无缝高效的静态网格 Actor 传输方法而设计。通过简单的复制和粘贴操作，您可以轻松交换详细的场景信息，从而简化您的关卡设计工作流。
