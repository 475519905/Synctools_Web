SyncTools for Unity - 使用文档

SyncTools Pro 安装与使用指南

📦 安装

Unity 设置

第一步：安装 FBX Exporter 包

  1. 打开 Unity 编辑器

  2. 进入 Window > 包管理器

  3. 点击左上角 + 按钮，选择 通过 git URL 添加包

  4. 输入：com.unity.formats.fbx

    • 或在 Unity Registry 中搜索 FBX Exporter 并安装

  5. 等待安装完成

第二步：安装 SyncTools 插件

  1. 将 SyncTools 文件夹复制到 Unity 项目的 资产s 目录中

  2. Unity 将自动编译。完成后，顶部菜单栏中将出现 SyncTools 菜单

第三步：配置设置

  1. 打开 SyncTools > 设置

  2. 设置缓存路径（缓存 路径）

    • 默认：文档s/cache

    • ⚠️ 重要：必须与 Blender/Maya 使用的缓存路径一致

  3. 设置导入文件夹（导入 文件夹）

    • 默认：Assets/SyncTools缓存/Incoming

    • 必须以 Assets/ 开头

🚀 快速入门

从 Unity 复制到 Blender/Maya

  1. 选择对象

    • 在 Unity 中，在层级面板中选择要导出的 Game对象

  2. 复制（导出选中内容）

    • 菜单：SyncTools > 复制 (导出 Selected)

    • 快捷键：Ctrl+Shift+C（Windows）或 Cmd+Shift+C（Mac）

  3. 完成

    • 导出的数据已保存至缓存目录

    • 现在可以在 Blender/Maya 端进行粘贴/导入

从 Blender/Maya 粘贴到 Unity

  1. 确认缓存路径一致

    • 确认 Unity 的缓存路径与 Blender/Maya 的相同

  2. 粘贴（导入最新内容）

    • 菜单：SyncTools > 粘贴 (导入 Latest)

    • 快捷键：Ctrl+Shift+V（Windows）或 Cmd+Shift+V（Mac）

  3. 完成

    • 最新的缓存内容被导入并实例化到场景中

    • 对象名称和层级结构自动恢复

⚙️ 设置参考

缓存路径（缓存 路径）

  • 用途：存储用于复制/粘贴交换的缓存文件

  • 默认：文档s/cache

  • 要求：必须与 Blender/Maya 使用的路径相同

导入文件夹（导入 文件夹）

  • 用途：导入的资产在 Unity 项目中的存放位置

  • 默认：Assets/SyncTools缓存/Incoming

  • 要求：必须以 Assets/ 开头

导入后删除缓存文件（删除 缓存 文件s After 导入）

  • 用途：粘贴成功后自动清理缓存文件

  • 默认：启用

  • 建议：保持启用，防止缓存文件夹不断增大

设置导入摄像机近裁剪面（设置 导入ed Camera Near Clip）

  • 用途：自动为导入的摄像机设置近裁剪面

  • 默认：启用，数值为 1.0

  • 使用场景：避免 Unity 中过小的近裁剪面值导致的渲染问题

⌨️ 快捷键

操作 Windows Mac
复制 Ctrl+Shift+C Cmd+Shift+C
粘贴 Ctrl+Shift+V Cmd+Shift+V

自定义快捷键：Unity 菜单 编辑 > Shortcuts

📋 工作流示例

场景一：Unity → Blender

1. Unity：选择对象 → 复制（Ctrl+Shift+C）
2. Blender：打开目标场景 → 粘贴操作
3. 完成：对象已导入 Blender

场景二：Maya → Unity

1. Maya：选择对象 → 复制操作
2. Unity：打开目标场景 → 粘贴（Ctrl+Shift+V）
3. 完成：对象已导入 Unity 场景

场景三：迭代循环

1. Unity → Blender（编辑材质）
2. Blender → Unity（预览效果）
3. Unity → Blender（继续调整）
...根据需要重复

⚠️ 故障排除

问题一：复制失败，提示缺少 FBX Exporter

解决方案：

  • 确认 FBX Exporter 包已安装

  • 在 包管理器 中搜索并安装 com.unity.formats.fbx

问题二：粘贴失败，找不到缓存文件

解决方案：

  • 确认缓存路径与 Blender/Maya 一致

  • 确保已在 Blender/Maya 端执行了复制/导出操作

  • 检查缓存文件夹中是否存在 export_*.fbx 文件

问题三：导入的对象名称不正确

解决方案：

  • 确认缓存目录中存在 hierarchy.txt

  • 检查该文件是否被其他进程锁定或已损坏

问题四：导入后材质丢失

解决方案：

  • 检查文件内的材质/贴图路径

  • 确保贴图文件位于预期目录中

  • 在 Unity Inspector 中检查 FBX 导入的材质设置

📌 重要说明

  1. ✅ 缓存路径必须一致（Unity、Blender、Maya 三端）

  2. ✅ Unity 端必须安装 FBX Exporter

  3. ✅ 命名规则：避免使用特殊字符（如 @#%）

  4. ✅ 版本一致性：在所有平台使用相同版本的 SyncTools

  5. ✅ 支持撤销：所有操作均支持 Unity 撤销（Ctrl+Z）
